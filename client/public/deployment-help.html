Vercel Deployment Help

Common Error: '@vercel/next' Source Error

Error Message

Error: Specified "src" for "@vercel/next" has to be "package.json" or "next.config.js"

Solution

This error occurs when Vercel can't find a proper Next.js configuration. To fix:

Option 1: Deploy directly from client directory

The simplest solution is to deploy directly from the client directory:

  • Using CLI: cd client && vercel
  • Using Vercel Dashboard: Set Root Directory to client

Option 2: Update vercel.json

If you must deploy from the repository root, ensure your root vercel.json has proper configuration:

{
  "buildCommand": "cd client && npm install && npm run build",
  "outputDirectory": "client/.next",
  "framework": "nextjs"
}

Required Environment Variables

  • DATABASE_URL: PostgreSQL connection string
  • Or individual database parameters: PGHOST, PGPORT, PGUSER, PGPASSWORD, PGDATABASE
  • ADMIN_KEY: Admin access key
  • SESSION_SECRET: Session encryption key

After Deployment

Once deployed, check the health endpoint at /api/health to verify your deployment is working correctly.
